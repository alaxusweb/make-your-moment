# 失敗防止の要点

| 避けること | 理由 | 正しい対応 |
|---|---|---|
| `7×5`を物理寸法と解釈 | これは配置数 | 7列×5行＝35枠として扱う |
| 1536×1024を2倍化 | 線や質感は増えない | 高解像度個別素材から直接組版 |
| 4枚生成を境界で等分 | 欠け・隣画像混入 | 重複抽出で検査、分離不能なら単品再生成 |
| 最大連結成分だけで除去 | 白枠接続時に異物も残る | 暗背景目視を必須にする |
| 画像生成に文字を書かせる | 誤字・崩れが発生 | manifestから決定的に組版 |
| PNGとPDFを別々に配置 | 座標と内容がずれる | PDFを先に作り、そのPDFをPNG化 |
| PDF背景に旧花名を残す | 二重文字になる | 白紙ラベルをラスタ層へ描き、花名はPDFテキストのみ |
| 表題まで編集可能にする | 校了デザインが変わる | 花名だけを編集可能にする |
| QAファイルを販売物へ入れる | ガイド線が商品に混入 | `customer/`と`qa/`を分離 |
| OSフォント本体をコピー | 再配布条件の問題 | システムフォントを参照し、ファイルは同梱しない |

停止条件：欠け、異物、マゼンタ、35枠不一致、中心ずれ、透過不良、PDF/PNG差、
花名以外の編集可能文字、QAガイド混入のいずれかが残る場合は完成としない。
